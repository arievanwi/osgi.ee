Factory components that can be used to create either transaction managed or normal data sources by means of a factory configuration in CM. To automatically construct a data source:
  • Create a configuration with service.factoryPid set to "datasource" (for normal data sources) or "XAdatasource" for transaction managed data source
  • Set the properties for this data source via normal settings. The properties should be looked up in the source, but for clarity some are given:
    • jdbc.url, jdbc.driver, jdbc.user, jdbc.password. The various settings for the JDBC connection.
    • pool.idle.min, pool.idle.max, pool.active.max, pool.wait. The pool parameters.
    • validation.query, validation.timeout. The validation query and query time-out
Requires an OSGi SCR.
